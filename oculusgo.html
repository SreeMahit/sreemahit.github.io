Oculus Go: Onboarding
First Time Users

Role Research, Ideation
and Video Prototyping
Team Solo Project
Duration 4 Weeks,
Winter 2019

The Problem

The city 's planetarium is setting up a new VR Zone for school students from grades 8-12. Most of these students have never used a VR Headset before and need to be onboarded before they can start using the headset and its controller.


The Solution

A set of stages to onboard these students was designed, with the intention of creating and engaging and playful gameplay. The onboarding covers 5 consecutive steps, from putting the headset on to learning how to use the controller.



Key Stages

The proposed onboarding covers five stages.


01 Headset Adjustment

02 Controller Callibration

03 Trigger Control

04 Touchpad Control

05 Exit Control

Storyboard

The major parts of the final storyboard that was developed.

Headset Adjustment

The user adjusts the headset by adjusting the straps and the lenses for a comfortable fit.



Controller Callibration

To calibrate the controller to the dominant hand, the user raises her hand and the controller reads the same.



Trigger Control

The user learns how to use the trigger and removes objects around her by selecting them.



Trackpad Control

The user learns to use the trackpad to zoom in and out of the environment.



Exit Control

The user then exits the onboarding environment by using the back button on the controller.



The Process

The first step in this project was to jot down the design objectives that helped me design the onboarding. This was done keeping in mind the target audience, 13-17 year old school students who haven't used this technology before.

01

The onboarding should be engaging and playful, with the steps being interactive.

02

The steps should be familiar and intutitive, making the controls easy to learn.

03

The user should be discouraged from moving around, as the space is constrained.

04

The entire sequence should be under 3 minutes as the user has limited time with the headset.


Initial Direction

With an understanding of what is required, I storyboarded one version of the various steps of the onboarding. This iteration focuses on a gamified approach where every step is compelted while the user is playing a game.


Drawbacks

While this version is very engaging, it has drawbacks which needed to be rectified.

01

It distracts the user from learning how to use the controls itself.

02

It instills a certain mental model which isn’t applicable to other applications.

03

The limited time frame would disrupt gameplay, leading to a poorer experience.

04

Improper mapping between actions and functions would confuse the any first-time user.

Considering the drawbacks, I ideated over a refined version addressing the issues and incorporating improved gameplays.

Refinded Version


The onboarding was separated into five different segments, each segmented focusing on one button and one subsequent interaction for practice.

Input Interactions

I jotted down the various input interactions before proceeding to developing the storyboards.

The headset uses a tracked hand controller, namely the Oculus Go Controller.


Of all the input options available, this onboarding focuses on three options, the trigger, touchpad and back.


The onboarding uses non-realistic interactions simulating how a child would interact in a space-based VR application.


User Interactions

The next step was to highlight what buttons of the controller would be included in the onboarding, keeping in mind the target audience and usage.

The trigger is used to select objects in the virtual environment


The touchpad is used to zoom into the virtual environment


The back button is used to exit the current stage of the virtual environment


User Journey

The final step was to design the various steps of the onboarding, stage after stage. The storyboard itself can be found below.



The user is prompted to check if the headset is comfortable.


To callibrate the controller, the user is asked to lift her dominant hand.


The controller senses the hand being held up.


The controller appears in the virtual hand, thus signalling a completion in callibration.


A virtual character guides the user through onboarding.


To practice selecting objects, the user is prompted to select the objects in front of her.


The user can select as many objects as she wants, removing them from the environment.


The character motivates the user with encouraging messages.


In the next step, the user learns how to use the trackpad.


The user is asked to select an object first.


The user then zoom closer to the object using the trackpad.


The user can move from one object to another by simply zooming into them.


The last step is to learn to exit the environment, using the back button


The environment slowly fades out, signaling the end of the turorial.



Learning

This project included designing for extended realities, designing for children and designing for real-life use cases. Working with new technologies like extended reality brough with it a unique set of challenges. It also taught me how to design for first-time users, where mental models can't be taken for granted.


Scope

The project can be taken further by developing a working prototyping on Unity 3D and conducting usability testing with students of that age-group. Since this deliverable was meant for a planetarium, it would be important to evaluate how it would work out in the constraints provided by the planetarium itself.
